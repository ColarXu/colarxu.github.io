写给文科生的 Command Line 教程 #1

2022.04.15

你肯定在电影上看过，黑客是怎样用电脑的。

跟我们拿着鼠标到处点点点不一样，他们通常会对着一个满是代码的屏幕，疯狂地敲击键盘。

这种「使唤」电脑的方式，便涉及到本文要讲的 Command Line。

什么是 Command Line？

Command Line，也就是「命令行」，一行一行命令的意思。

这些命令，指的是我们给计算机下的指令。比如点播放音乐、打开文件夹、放大图片、关闭浏览器、退出软件、清空回收站……这些都是「命令」。

电脑是个很聪明的玩意，但同时也很呆板，跟它们的祖先算盘一样——拨一下，才会动一下。如果没有人给电脑下命令，它们就会翘着脚，啥事都不干。

我们用鼠标和手指在屏幕上点这点那，便是在「指挥」电脑干这干那。近些年兴起的智能语音助手，比如 Siri、Cortana、Alexa、更是可以直接「喊」电脑来帮我们做事。

而 Command Line，这种最古早的命令手段，比起上两种方式则要麻烦得多，它需要我们给电脑「发短信」——在键盘上打一串文字，告诉电脑干啥干啥。且由于电脑不会说人话，所以打字的时候还得使用「计算机语言」，也就是「代码」，电脑才会明白你想要它干嘛。

Command Line 有什么用？

既然如此麻烦，为什么还要学 Command Line 呢？

首先必须得承认，如果你不是开发者，学习 Command Line 基本没啥用。就像不读柏拉图，不读孔老庄，依然可以毫无阻碍地过完自己的一生。

但如果你对编程感兴趣，对计算机运行的原理感兴趣，或是想在疫情封闭期间，找根棍子来搅动自己死水一般的大脑的话，学习 Command Line 可能会是一种乐趣。一旦掌握了这种命令手段，你会发现，当批量处理文件时，用 Command Line 确实比用鼠标拖拽方便得多。

两年前我搭了个博客，每次发布文章，都需要敲几行代码，这是我跟 Command Line 接触的源起。现在这篇文章，主要也是写给自己——当下次忘掉 Command Line 怎么用时，知道去哪里重新捡起来。

如果有人看了这篇文章，并学会 Command Line 的一些基本操作，那便是我的荣幸了。

准备工作

好了，在给电脑「发命令」之前，我们也得知道去哪里发送才对。

以 Mac 为例，系统里专门有一个软件来做这件事，它的名字叫 Terminal（终端）。

有两种方式可以找到它。

方法一：打开 Lauchpad（启动台），在「其他」中，找到 Terminal。

方法二：用 Spotlight（聚焦） 直接搜索 Terminal。

这个黑不溜秋的 Terminal，就是能让我们跟电脑「聊天」的灵媒。

点开 Terminal，开始探索计算机的内部世界吧。
